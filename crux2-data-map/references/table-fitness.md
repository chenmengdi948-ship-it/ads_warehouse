# 表适用性评估工作流（table-fitness）

**触发**：用户有具体业务需求，评估候选表能否满足。

**步骤**：
1. `table-detail --detail-types BASIC` → 表类型、数据模型、生命周期
2. `table-detail --detail-types COLUMN` → 对比需求字段是否存在
3. `table-detail --detail-types PARTITION` → 数据时间覆盖范围和更新频率

**LLM 评估输出格式**：
```markdown
| 维度 | 需求 | 实际 | 匹配度 |
|------|------|------|--------|
| 粒度 | 用户级 | 用户级 | ✅ 匹配 |
| 时效性 | 每日更新 | 每日 T+1 | ✅ 匹配 |
| 时间范围 | 近30天 | 至今2年 | ✅ 满足 |
| 字段覆盖 | dau/region | 均存在 | ✅ 满足 |
| 表质量 | — | 优质表+核心表 | ✅ 高质量 |

**结论**：该表适合用于此需求，建议使用 dtm 分区过滤最近30天数据。
```
